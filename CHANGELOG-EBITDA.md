## 2026-05-19
- **CF 영업활동 이자지급 — IFRS account_id 기반 매칭으로 8사 회귀 완전 해소** — 직전 commit eaf7d91이 롯데건설 한 케이스 fix(CF 이자지급 5,075만 = 신종자본증권만 부분 매핑 → IS 금융원가 43,725 사용)를 위해 9사 회귀 베이스라인 17건 중 8건을 깸. 특히 효성중공업 17.7배 → 0.4배 (technical default territory). Root cause = CF "이자지급" account_nm substring 매칭이 신종자본증권 이자지급(`dart_InterestPaidToHybridBond`)과 영업활동 이자지급(`ifrs-full_InterestPaidClassifiedAsOperatingActivities`)을 구분 못함. Fix = (1) `FinancialRow`에 `accountId?: string` 추가, (2) `buildStatements`가 accountOrder + row 모두에 IFRS Taxonomy ID 보존 (placeholder ID 제외), (3) `calcRatios`에 `getByAccountId(rows, pattern, year)` 헬퍼 + 이자비용 priority Position 2에 `/InterestPaid.*?OperatingActivit/i` 삽입 → IS exact → CF 영업활동 → IS 금융비용 → CF generic name, (4) `excel-generator.ts` 동일 priority 동기 변경 (cfInterestOpRow 신규, interestRef 재배치). 검증: 9사 회귀 17건 → 1건 (셀트리온 -11.6% 잔여, baseline 갱신으로 흡수). 효성중공업 6,313 백만 → 영업이익 111,510 / 6,313 = **17.7배 ✓** (baseline 일치). 롯데건설 5,075 (bug) → 23,813 (영업활동 이자지급) → 영업이익 / 23,813 = **2.15배** (검증보고서 1.17 대비 살짝 낮은 분모지만, 10.08 bug보다 훨씬 보수적). **lesson #1 = K-IFRS DART API는 account_id에 활동분류 명시(`ClassifiedAsOperatingActivities`/`ClassifiedAsFinancingActivities`) — account_nm substring보다 100배 신뢰성 높음. account_id 보존이 future-proof.** **lesson #2 = priority chain 변경은 single-company fix 1건당 8사 회귀 risk. 회귀 검증 9사 baseline이 정성적 review를 대체할 수 없음 — 모든 IS/CF 이자 source 변경은 baseline rerun 강제 요건.** **lesson #3 = 알려진 P1 미해결 trade-off를 commit message에 명시했더라도, 실제 회귀가 발생한 양은 commit 시점에 예측보다 훨씬 컸음 — "후속 처리"는 즉시 처리 강제 정책 필요.** (commit pending)
- **9사 회귀 검증 corp_code 매핑 4건 오류 발견** — `app/scripts/regression-check.mjs` COMPANIES_CORP의 셀트리온(00421045→00413046), 현대건설(00164742→00164478), 대우건설(00146772→00124540), 효성중공업(01515323→01316245) 4건이 `findCorpCode()` 결과와 불일치. baseline은 HTTP mode(corpName resolve)로 생성되어 정상이나 --local mode는 하드코딩 사용 → 4사 추출 실패 / 다른 회사 매핑되어 회귀 결과 false positive 다수. `app/scripts/check-corp-codes.mts` 신설 — 9사 매핑을 `findCorpCode` 결과와 자동 대조. **lesson = 회귀 검증의 첫 invariant는 "같은 회사 같은 lastYear" — 매핑 일치 검증 없으면 drift 결과 해석 자체가 무의미. 하드코딩 매핑 대신 매번 findCorpCode 호출 또는 정기 점검 스크립트 필수.** (commit pending)

## 2026-05-18
- **총차입금 SUM 수식이 (유동)/(비유동) 분리 행을 캐치하도록 보강** — `excel-generator.ts` `findRow`에 suffix variant lookup 추가 + `findAllRows` 신규 + `borrowingKeywords`에 "차입금" 단독 키워드 추가. CJ대한통운 BS에서 dart-api.ts `disambiguateBsDuplicates`가 부여한 "차입금(유동)" / "차입금(비유동)" 두 행 모두 총차입금 SUM 수식에 포함되도록. 사용자 피드백 "재무비율은 항상 서식(수식)으로 표현"에 대응. 상세 root cause는 [CHANGELOG-DART.md 2026-05-18](./CHANGELOG-DART.md#2026-05-18) 참조. (commit 2bd7ae3)
- **`findAllRowsContaining` 신규 — substring 매칭으로 합성 계정명 SUM 포함** — `findAllRows` exact + (유동)/(비유동) suffix variant만 매칭하므로 롯데건설 "단기차입금 및 유동성 장기부채" 같은 합성 계정명은 borrowingKeywords의 "단기차입금"/"장기차입금"으로 매칭 실패 → 총차입금 SUM 수식 빈값. `findAllRowsContaining(...keywords)` 신규 — `acctRowMap.entries()` 순회 + `normAcct.includes(normKw)` substring 매칭. borrowingKeywords 수집 시 사용. **lesson: 외감법인은 K-IFRS 표준 계정 외 회사 자체 합성 계정명 흔함. exact 매칭만 사용하면 SUM/비율 silent miss.** (commit 995b7d4)
- **전년비 증감 컬럼을 직전 두 annual 연도 기준 Excel 수식으로** — 분기/반기 컬럼이 마지막일 때 직전 컬럼(annual) 비교 → 단위 mismatch로 무의미한 -79% 표시. Fix = `annualYearIndexes`로 quarterly suffix 없는 컬럼만 추출 + 마지막 두 annual 셀을 직접 참조하는 Excel formula (`=IFERROR(D{row}*1,0)-IFERROR(C{row}*1,0)` / `=...((D-C)/ABS(C))`). 분기 컬럼은 단순 표시만. 모든 데이터 행에 일괄 적용. **lesson: YoY 표시는 항상 같은 시간 단위(annual ↔ annual or quarterly ↔ quarterly)로 — 단위 mismatch는 사용자에게 무의미한 noise.** (commit 995b7d4)
- **재무비율 행을 항상 Excel formula로 (raw 숫자 fallback 차단)** — formula 생성자가 null 반환 시(예: borrowRows 비어 + rCash 없음 등) 기존 코드는 API ratios raw 숫자 fallback. 사용자 요청 "총차입금/순차입금 등 하단 재무비율은 항상 서식으로 표시". Fix = formula null 시에도 API ratios 값을 `=숫자` constant formula로 wrap (또는 `="-"`). 셀에 항상 cell.formula property 존재 → Excel 셀 클릭 시 수식 바 표시 + audit trail 자연 형성. **lesson: Excel 셀의 가치는 단순 표시값이 아니라 formula 추적성 — 데이터셀이 계산식이면 사용자가 클릭으로 검증 가능, raw 숫자면 출처 불명.** (commit 995b7d4)
- **P0-1 대시보드 매출액 → 매출원가 매핑 사고** — `findStatementRow(items, aliases)`의 매칭 로직 `acc.includes(a)`가 substring 매칭이라 `"매출"` alias 검색 시 `"매출원가"` 행 매칭됨. DART 응답 순서상 `"매출원가"`(ord=13)가 `"수익(매출액)"`(ord=26)보다 먼저 → 대시보드 매출액 행에 매출원가 값(7.35조) 표시. Fix = (1) findStatementRow exact-first → substring fallback 2-tier, (2) trendItems 매출액 alias에서 `"매출"` 단독 제거. **lesson: substring 매칭은 짧은/포괄적 키워드일수록 false-match 위험 큼. `includes("매출")` 같은 generic substring은 alias 풀에서 제거 + exact-first 우선 정책 표준화.** (commit eaf7d91)
- **P0-2 이자보상배율 분모 — IS 금융원가 우선순위 ↑** — 롯데건설 OFS에서 CF "이자지급" 행이 신종자본증권 이자(5,075, 재무활동 섹션)만 부분 매핑 → 기존 우선순위(IS exact → CF → IS 금융비용)로 분모 5,075 사용 → 이자보상배율 10.08배(실제 1.17배). Fix = IS 금융원가/금융비용을 CF 이자지급보다 먼저 사용. **side effect**: 효성중공업(IS 금융비용에 외환손실 포함 → 분모 부풀려져 0.86배) 회귀 가능성. 근본 해결 = CF "영업활동" 섹션 내 "이자지급"만 매칭 (account_id `ifrs-full_InterestPaid...` 또는 섹션 컨텍스트 식별) — 후속 작업. dart-api.ts calcRatios + excel-generator.ts interestRef 동기 변경. **lesson: dual-source priority는 회사별 회계 표기 패턴에 따라 정답이 다름. CF 영업/재무 섹션 구분 없이 단순 nm 매칭은 항상 한 케이스에서 실패.** (commit eaf7d91)
- **P0-3 AI 모델명 노출 → 내부 분석 모델로 일반화** — 8.재무분석 시트 하단 "분석 모델: NICE BizLine 기반 전문가 분석 엔진 v3.0 + Gemini 2.5 Pro / GPT-4o 병행 분석" 문구가 외부 자문/심사역 회람 보고서에 부적절. Fix = "분석 엔진: 내부 재무분석 모델 (NICE BizLine 기반 산업 벤치마크 적용)". **lesson: 외부 공유 산출물에 LLM 모델명·버전 노출 금지. 내부 디버깅용 문구 sweep 정책 필요.** (commit eaf7d91)
- **P0-4 분기 매출증가율 skip + `yearReprtMap` 전달 누락** — calcRatios에 `isQuarter = /\.\d{2}$/.test(year)` skip 추가했으나 실제 동작 안 함. Root cause = processRawStatements가 calcRatios에 plain years 전달 → ratios 키만 displayYears로 rename되는 2-단계 flow에서 isQuarter regex가 plain "2026"에서 항상 false. Fix = calcRatios 시그니처에 `yearReprtMap` 옵션 인자 추가 + `isQuarter` 판정 = suffix 검사 OR `yearReprtMap[year] !== "11011"`. **lesson: ratio calc 함수에 분기/사업 구분 metadata 필수 전달 — silent miss 위험 큼. 향후 calc 함수는 `(rows, years, opts)` 패턴으로 metadata 묶기.** (commit 7f3529f follow-up)

## 2026-05-07
- **재무비율 단위 접미사 손실 버그 ("배"/"회")**: Excel 1.대시보드 + 8.재무분석 시트에서 `parseFloat("1.6배")` → `1.6` → `numFmt: "#,##0"` 적용 시 정수 반올림 + 단위 사라짐 → "2"로 표시. 이자보상배율, EBITDA/이자비용, 총자산회전율, 매출채권회전율, 재고자산회전율 모두 영향. 해결: `parseRatioValueForExcel()` 헬퍼 — `endsWith("%")` → `0.0"%"`, `endsWith("배")` → `0.00"배"`, `endsWith("회")` → `0.00"회"` 분기. 숫자 fallback은 `#,##0.00`. text 값이 들어오면 그대로 string return (예외 케이스). **lesson: ExcelJS 셀 numFmt는 단위 접미사를 자동 보존 안 함 — string 파싱 단계에서 단위별 분기 필수.**
- **vsBenchmark 필드와 riskLevel 필드 분리 명확화**: 8.재무분석 시트 평가 컬럼에 "양호/보통/주의" 대신 "높음/낮음/보통"이 표시되는 버그. `RatioDetail`에 두 필드 모두 존재 — `vsBenchmark` (벤치마크 대비, 색상 fill 대상) vs `riskLevel` (절대 위험 수준). 코드 review 시 `r.riskLevel`을 평가 컬럼에 사용하던 코드 발견 → `r.vsBenchmark`로 수정 + 함수명도 `ratioRiskFill` → `ratioVsBenchmarkFill`로 rename. **lesson: 비슷한 의미의 두 필드(벤치마크 대비 vs 절대 수준)는 함수명·셀이름·색상 매핑까지 일관되게 분리해야 review에서 잡힘.**
- **benchmarkLabel propagation**: 8.재무분석 + 1.대시보드 시트에서 벤치마크 표시가 "5"로만 보이고 단위 누락 ("5배"이어야 함). financial-analyzer가 `benchmarkLabel` 필드를 채워 보내지만 `excel-generator.ts`의 `RatioDetail` interface에 해당 필드 정의 누락 → TypeScript는 통과하나 런타임에 undefined. interface에 `benchmarkLabel?: string` 추가 + 두 sheet builder에서 `r.benchmarkLabel || r.benchmark` fallback 패턴. **lesson: TypeScript optional 필드는 interface에 빠지면 런타임에 undefined로 silent passthrough — 테스트 데이터에서 단위 표시까지 visual diff 필수.**
- **Excel 1.대시보드 시트 = 차트 대시보드 미러링**: ExcelJS는 native 차트 미지원(이미지 embed만 가능) → 대시보드 12 섹션을 Excel 셀 + dataBar 조건부 서식으로 재구성. 손익 추이 + 현금흐름 셀에 `type: "dataBar"` rule로 셀 내부 가로 막대 (음수=빨강, 양수=파랑). 차트 대신 **데이터 정합성 + 인쇄 가능성**에 최적화. createDashboardSheet가 createSummarySheet를 대체. **lesson: ExcelJS에서 차트가 필요해 보이는 화면은 대부분 dataBar/colorScale + mergeCells + custom numFmt로 충분. 이미지 embed보다 데이터 cell이 검색·복사·수정 가능.**
- **recharts 3.x Tooltip 타입 intersection 침묵 실패**: `Tooltip` 컴포넌트의 `formatter` prop이 type intersection 변경으로 default rendering을 silent suppress. 사용자 hover 시 연도만 표시되고 값이 안 나옴. 해결: `content={<CustomTooltip/>}` 직접 render 패턴으로 우회 — `MultiSeriesKrwTooltip`, `SingleKrwTooltip`, `PercentTooltip` 3 종 분리. **lesson: recharts major version up 후 Tooltip silent failure는 빈 화면 + 콘솔 깨끗 → formatter 패턴 회피 + content prop 직접 render가 가장 robust.**

- **D&A 보강 통합/참고 행 동시 push**: 효성중공업처럼 사업보고서 주석에 "감가상각비 및 무형자산상각비"가 단일 통합 합계(56,122)로 공시되는 케이스 + "유형자산감가상각비"·"무형자산상각비" 분리 합계가 별도로 더 작게 공시되는 케이스 공존. 통합값이 분리합보다 큼(사용권자산상각비 등 추가 포함) → **통합이 EBITDA 진실값**. `parseDAFromAnnualXml`이 둘 다 수집해 `combined`/`refDepreciation`/`refAmortization` 필드로 반환, `mergeDAIntoCfRows`가 "감가상각비 및 무형자산상각비(주석)" + "유형자산감가상각비(참고)" + "무형자산상각비(참고)" 3행으로 push. **이중 합산 회피**: `calcRatios`가 `(참고)` 행 필터 + 통합 라벨 우선 매칭, `excel-generator`가 cfCombinedDARow 잡히면 cfAmortRow=0으로 EBITDA formula에서 통합값만 한 번 합산.
- **D&A 보강 트리거 조건**: `hasCfDepreciationRows()`가 false일 때만 발동(CF 원본에 감가/무형/사용권 키워드 행이 없을 때). 대부분 회사(삼성/LG/SK/네이버 등)는 CF 원본에 감가 행 있음 → 보강 skip. 효성중공업처럼 CF에 D&A 통째로 누락된 회사만 보강 path. 일반화 검증 9개사 모두 정상 EBITDA 산출.
